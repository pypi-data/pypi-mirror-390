## 1. Change Summary

## 3. Developer Documentation
- Update API documentation (function signatures, parameters, return values)
- Document new configuration options or environment variables
- Update architecture diagrams or design docs
- Add code examples or usage patterns

## 4. Follow-Up Documentation
- List related documentation files that should be reviewed
- Identify outdated sections that need updates
- Suggest new documentation pages if needed
- Note any README or changelog updates required
